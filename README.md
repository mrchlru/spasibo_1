# HR Spasibo Bot - Система внутренней мотивации сотрудников

## 📋 Описание приложения

**HR Spasibo Bot** — это комплексная система внутренней мотивации сотрудников, построенная на базе Telegram WebApp. Приложение позволяет сотрудникам получать, тратить и передавать внутреннюю валюту "спасибки", участвовать в лотереях, покупать товары в корпоративном магазине и взаимодействовать с коллегами.

### 🎯 Основные возможности

- **Система спасибок** - внутренняя валюта для поощрения сотрудников
- **Корпоративный магазин** - покупка товаров за спасибки
- **Рулетка удачи** - азартные игры с призами
- **Переводы между сотрудниками** - отправка спасибок коллегам
- **Бонусная карта** - система лояльности с QR-кодами
- **Совместные подарки** - система приглашений на совместные покупки подарков
- **Таблица лидеров** - рейтинг пользователей с разными периодами и категориями
- **Бонусы Statix** - интеграция с бонусной системой Statix, обмен спасибок на бонусы
- **Онбординг** - интерактивные обучающие истории для новых пользователей
- **Мини-игра Dino Game** - развлекательная игра в приложении
- **Автоматические задачи** - планировщик для начисления бонусов на ДР, сброса лимитов и генерации баннеров
- **Административная панель** - управление пользователями и товарами
- **Статистика и аналитика** - детальная отчетность по активности
- **Система уведомлений** - интеграция с Telegram

## 🗄️ Структура базы данных

### Основные таблицы

#### Users (Пользователи)
- `id` - Уникальный идентификатор
- `telegram_id` - ID в Telegram
- `first_name`, `last_name` - Имя и фамилия
- `position` - Должность
- `department` - Отдел
- `status` - Статус (pending/approved/rejected/blocked)
- `balance` - Баланс спасибок
- `is_admin` - Права администратора
- `ticket_parts` - Части билетов для рулетки
- `tickets` - Готовые билеты
- `card_barcode` - QR-код бонусной карты
- `has_seen_onboarding` - Прошел ли обучение

#### Transactions (Транзакции)
- `id` - Уникальный идентификатор
- `sender_id` - ID отправителя
- `receiver_id` - ID получателя
- `amount` - Сумма перевода
- `message` - Сообщение к переводу
- `timestamp` - Время операции

#### MarketItems (Товары магазина)
- `id` - Уникальный идентификатор
- `name` - Название товара
- `description` - Описание
- `price` - Цена в спасибках
- `price_rub` - Цена в рублях
- `stock` - Количество на складе
- `original_price` - Старая цена (для скидок)
- `is_auto_issuance` - Автоматическая выдача
- `is_archived` - Архивирован ли товар
- `is_shared_gift` - Флаг совместного подарка

#### StatixBonusItems (Бонусы Statix)
- `id` - Уникальный идентификатор
- `name` - Название товара
- `description` - Описание
- `image_url` - Ссылка на изображение
- `is_active` - Активность товара
- `thanks_to_statix_rate` - Курс валют (спасибки за 100 бонусов)
- `min_bonus_per_step` - Минимум бонусов за шаг
- `max_bonus_per_step` - Максимум бонусов за шаг
- `bonus_step` - Шаг увеличения

#### SharedGiftInvitations (Приглашения на совместные подарки)
- `id` - Уникальный идентификатор
- `buyer_id` - ID покупателя
- `invited_user_id` - ID приглашенного пользователя
- `item_id` - ID товара
- `status` - Статус (pending/accepted/rejected/expired)
- `created_at` - Время создания
- `expires_at` - Время истечения
- `accepted_at` - Время принятия
- `rejected_at` - Время отклонения

#### Purchases (Покупки)
- `id` - Уникальный идентификатор
- `user_id` - ID покупателя
- `item_id` - ID товара
- `timestamp` - Время покупки

#### ItemCodes (Коды товаров)
- `id` - Уникальный идентификатор
- `code_value` - Код или ссылка
- `is_issued` - Выдан ли код
- `market_item_id` - ID товара
- `issued_to_user_id` - ID получателя

#### UserSessions (Сессии пользователей)
- `id` - Уникальный идентификатор
- `user_id` - ID пользователя
- `session_start` - Начало сессии
- `last_seen` - Последняя активность

## 🚀 Установка и запуск

### Предварительные требования
- Python 3.8+
- Node.js 16+
- PostgreSQL 12+
- Telegram Bot Token

### Backend

1. **Установка зависимостей:**
```bash
cd backend
pip install -r requirements.txt
```

2. **Настройка переменных окружения:**
Создайте файл `.env` в папке `backend/`:
```env
DATABASE_URL=postgresql://user:password@localhost:5432/hr_spasibo
ADMIN_API_KEY=your_admin_api_key
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id
TELEGRAM_ADMIN_IDS=admin_id1,admin_id2
TELEGRAM_ADMIN_TOPIC_ID=your_topic_id
TELEGRAM_PURCHASE_TOPIC_ID=your_purchase_topic_id
TELEGRAM_UPDATE_TOPIC_ID=your_update_topic_id
TELEGRAM_ADMIN_LOG_TOPIC_ID=your_log_topic_id
```

3. **Запуск сервера:**
```bash
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

4. **Запуск Telegram бота:**
```bash
python run_polling.py
```

### Frontend

1. **Установка зависимостей:**
```bash
cd frontend
npm install
```

2. **Запуск в режиме разработки:**
```bash
npm run dev
```

3. **Сборка для продакшена:**
```bash
npm run build
```

## 🔧 API Endpoints

### Пользователи
- `POST /users/auth/register` - Регистрация
- `GET /users/me` - Получить свой профиль
- `PUT /users/me` - Обновить профиль
- `GET /users/` - Список пользователей (админ)

### Транзакции
- `POST /transactions/send` - Отправить спасибки
- `GET /transactions/history` - История операций

### Магазин
- `GET /market/items` - Список товаров
- `POST /market/purchase` - Покупка товара
- `GET /market/statix-bonus` - Получить настройки бонусов Statix
- `POST /market/statix-bonus/purchase` - Купить бонусы Statix

### Совместные подарки
- `POST /shared-gifts/invite` - Создать приглашение на совместный подарок
- `GET /shared-gifts/invitations/{user_id}` - Получить приглашения пользователя
- `POST /shared-gifts/accept` - Принять приглашение
- `POST /shared-gifts/reject` - Отклонить приглашение
- `POST /shared-gifts/cleanup` - Очистить истекшие приглашения (админ)

### Рулетка
- `POST /roulette/spin` - Крутить рулетку
- `POST /roulette/assemble` - Собрать билет
- `GET /roulette/history` - История выигрышей

### Таблица лидеров
- `GET /transactions/leaderboard/` - Получить рейтинг
- `GET /transactions/leaderboard/my-rank` - Получить свою позицию
- `GET /transactions/leaderboard/status` - Получить статус доступности рейтингов

### Админ панель
- `GET /admin/stats` - Общая статистика
- `GET /admin/users` - Управление пользователями
- `POST /admin/items` - Управление товарами
- `GET /admin/banners` - Управление баннерами
- `GET /admin/statix-bonus` - Получить настройки бонусов Statix (админ)
- `PUT /admin/statix-bonus` - Обновить настройки бонусов Statix (админ)
- `POST /admin/generate-leaderboard-banners` - Генерировать баннеры лидерборда

### Планировщик задач
- `POST /scheduler/run-daily-tasks` - Выполнить ежедневные задачи (требует секретный ключ)
- `POST /run-monthly-tasks` - Выполнить ежемесячные задачи (требует секретный ключ)

## ⚙️ Автоматические задачи (Scheduler)

Система включает планировщик задач для автоматизации рутинных операций:

### Ежедневные задачи (`/scheduler/run-daily-tasks`)
- **Начисление бонусов на день рождения** - автоматическое начисление спасибок пользователям в день их рождения
- **Сброс билетов рулетки** - ежедневная проверка срока хранения и обнуление:
  - частей билетов, если прошло 3 месяца с последнего сброса;
  - готовых билетов, если прошло 4 месяца с последнего сброса.
- **Сброс лимитов переводов** - обнуление ежедневных счетчиков переводов

### Ежемесячные задачи (`/run-monthly-tasks`)
- **Генерация баннеров лидерборда** - автоматическое создание баннеров с топ-пользователями:
  - Топ-10 получателей спасибок за прошлый месяц
  - Топ-10 самых щедрых пользователей за прошлый месяц

### Настройка cron-задач

Для автоматического выполнения задач настройте cron:

```bash
# Ежедневно в 00:00 UTC
0 0 * * * curl -X POST https://your-api.com/scheduler/run-daily-tasks -H "X-Cron-Secret: your_admin_api_key"

# Ежемесячно 1-го числа в 00:00 UTC
0 0 1 * * curl -X POST https://your-api.com/run-monthly-tasks -H "X-Cron-Secret: your_admin_api_key"
```

## 🎁 Совместные подарки

Система позволяет пользователям покупать подарки совместно:

1. **Создание приглашения** - покупатель может пригласить коллегу для совместной покупки подарка
2. **Приглашение через Telegram** - приглашенный пользователь получает уведомление в Telegram
3. **Принятие/отклонение** - приглашенный может принять или отклонить приглашение
4. **Автоматический возврат** - при отклонении или истечении срока приглашения средства автоматически возвращаются покупателю
5. **Срок действия** - приглашения имеют срок действия (по умолчанию 7 дней)

Товары помечаются флагом `is_shared_gift` для включения функции совместных подарков.

## 🎮 Бонусы Statix

Интеграция с бонусной системой Statix позволяет пользователям обменивать спасибки на бонусы:

- **Настраиваемый курс** - администратор может установить курс обмена (по умолчанию: 10 спасибок = 100 бонусов)
- **Ползунок выбора** - удобный интерфейс для выбора количества бонусов (от 100 до 10000 с шагом 100)
- **Автоматический расчет** - стоимость рассчитывается автоматически по формуле: `(количество_бонусов / 100) * курс`
- **Управление в админке** - полный контроль над настройками товара через административную панель

Подробнее см. файл `STATIX_BONUS_IMPLEMENTATION.md`.
